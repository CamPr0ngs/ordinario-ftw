¿Cómo funciona el proyecto?

Una base de datos táctica pensada para buscar y filtrar la información básica de los personajes de Marvel Rivals al instante, usando tecnologías nativas del navegador y sin depender de ningún framework externo.

¿Por qué elegí este tema?

La elección de Marvel Rivals como el núcleo de esta base de datos táctica no fue una casualidad, sino una decisión completamente deliberada para potenciar el proceso de aprendizaje. Creo firmemente que la mejor manera de dominar, no solo el desarrollo web sino cualquier cosa, es trabajando sobre un entorno que realmente nos entusiasme, divierta y concumamos en nuestro día a día. Al trasladar los conceptos de la EE a un videojuego que sigo de cerca, el flujo de desarrollo dejó de ser una simple asignación escolar y se transformó en un proyecto que me daba gusto desarrollar, facilitándolo muchísimo.

Esa conexión directa con el tema cambia por completo el enfoque: como conozco el juego, me volví mucho más exigente con cosas como la interfaz y que se viera bien. Traté de no conformarme con que el código "solo funcione", sino que busqué que los colores hicieran match con la identidad visual del título, que los resplandores reaccionaran de forma limpia y que la separación de roles respete la lógica competitiva real. Al final, este proyecto es el balance perfecto entre cumplir con una arquitectura técnica estricta y el gusto que le tengo al juego, demostrando que las herramientas de ingeniería pueden aplicarse para cualquier tipo de información digital.

Objetivo del Proyecto

La idea principal de esta aplicación es centralizar e interactuar con el catálogo de héroes de Marvel Rivals de forma dinámica. En lugar de estar recargando la página a cada rato, la interfaz te permite separar a los personajes en sus tres roles de combate (Vanguardias, Duelistas y Estrategas) al instante, haciendo que la consulta de habilidades e inteligencia de campo sea fluida y rápida.

Puntos clave del desarrollo

  • Todo corre en el cliente: El procesamiento de los datos, el manejo del árbol de nodos y el renderizado en pantalla se hacen directamente en el navegador con JavaScript nativo, asegurando que la página responda al segundo.
  • Datos separados del diseño: Toda la información de los héroes (XML) está completamente aislada de la vista (HTML5/CSS3). Esto permite agregar personajes o cambiar estadísticas desde afuera sin tener que tocar el código de la interfaz web.
  • Estructura limpia y validada: Se implementó un esquema de validación que revisa que los datos no se rompan, obligando al sistema a respetar reglas clave como IDs únicos para cada héroe, consistencia en los roles y rutas correctas para las imágenes.

1. Base de datos (XML)

Toda la información de los héroes está guardada en un archivo estructurado independiente.

  • Datos separados del diseño.
  • Nodos organizados por jerarquía.

2. Validación (DTD)

Asegura el orden estricto de los elementos para que la información no se rompa ni se duplique.

  • Obliga a usar IDs únicos.
  • Valida la estructura del documento.

3. Motor Dinámico (AJAX)

Carga el archivo XML de forma asíncrona usando JS nativo para actualizar el contenido en tiempo real.

  • Peticiones rápidas y limpias.
  • Filtrado interactivo sin recargas.

4. Diseño (CSS)

Estilizado modular basado en un diseño de tablero oscuro responsivo con efectos de iluminación neón.

  • Estructuras con Grid y Flexbox.
  • Componentes visuales ordenados.
